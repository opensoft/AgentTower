# Data Model: Event-Driven Routing and Multi-Master Arbitration

**Branch**: `010-event-routes-arbitration` | **Date**: 2026-05-16 | **Plan**: [plan.md](./plan.md)

This document specifies the FEAT-010 storage layer:
- The new `routes` SQLite table
- Three new columns + one partial UNIQUE index on FEAT-009's
  existing `message_queue` table
- The schema-version bump v7 → v8 and migration shape
- In-memory worker state (shared counters + degraded flag) used by
  the routing worker and heartbeat thread

## 1. SQLite table: `routes`

One row per operator-created subscription. Created by `route add`
(FR-001), updated by `route enable` / `route disable` (FR-009),
removed by `route remove` (FR-003), and read by the routing worker
on every cycle.

```sql
CREATE TABLE routes (
  route_id              TEXT NOT NULL PRIMARY KEY,  -- UUIDv4 (FR-001)
  event_type            TEXT NOT NULL,              -- closed set (FR-005)
  source_scope_kind     TEXT NOT NULL,              -- closed set: any | agent_id | role
  source_scope_value    TEXT NULL,                  -- shape depends on kind (FR-001 + Clarifications Q1)
  target_rule           TEXT NOT NULL,              -- closed set: explicit | source | role (FR-006)
  target_value          TEXT NULL,                  -- shape depends on target_rule
  master_rule           TEXT NOT NULL,              -- closed set: auto | explicit (FR-007)
  master_value          TEXT NULL,                  -- agent_id when master_rule=explicit
  template              TEXT NOT NULL,              -- rendered string template (FR-008)
  enabled               INTEGER NOT NULL DEFAULT 1, -- 0|1; default enabled
  last_consumed_event_id INTEGER NOT NULL DEFAULT 0, -- per-route cursor (FR-002, FR-012)
  created_at            TEXT NOT NULL,              -- ISO-8601 ms UTC
  updated_at            TEXT NOT NULL,              -- ISO-8601 ms UTC
  created_by_agent_id   TEXT NULL                   -- agt_* or 'host-operator' sentinel
);

CREATE INDEX idx_routes_created_at_route_id
  ON routes(created_at, route_id);
```

### Column-level invariants

| Column | Invariant |
|---|---|
| `route_id` | UUIDv4 generated by `uuid.uuid4()`. Format `xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx` where `y` is one of `{8,9,a,b}`. Distinct from FEAT-006's `agt_<12-hex>` agent_id format. |
| `event_type` | One of: `activity`, `waiting_for_input`, `completed`, `error`, `test_failed`, `test_passed`, `manual_review_needed`, `long_running`, `pane_exited`, `swarm_member_reported` (FR-005). |
| `source_scope_kind` | One of: `any`, `agent_id`, `role`. |
| `source_scope_value` | `NULL` when `kind=any`. `agt_<12-hex>` agent_id string when `kind=agent_id`. `role:<role>[,capability:<cap>]` when `kind=role` (Clarifications Q1). |
| `target_rule` | One of: `explicit`, `source`, `role`. |
| `target_value` | `NULL` when `target_rule=source`. Single `agt_<12-hex>` or label when `target_rule=explicit`. `role:<role>[,capability:<cap>]` when `target_rule=role`; role MUST be in `{slave, swarm}` (FR-006). |
| `master_rule` | One of: `auto`, `explicit`. |
| `master_value` | `NULL` when `master_rule=auto`. `agt_<12-hex>` when `master_rule=explicit`. |
| `template` | UTF-8 string ≤ 4 KiB; placeholders match `\{<field>\}` where field is in the FR-008 whitelist. |
| `enabled` | `0` or `1`; default `1`. |
| `last_consumed_event_id` | `INTEGER ≥ 0`. Initialized to `MAX(events.event_id) OR 0` at insert time (FR-002). Monotonically non-decreasing during the route's lifetime (FR-012). Frozen while `enabled=0`. |
| `created_at`, `updated_at` | ISO-8601 millisecond UTC string from `routing.timestamps.now_iso_ms_utc()`. |
| `created_by_agent_id` | `NULL` if caller identity unknown. `agt_<12-hex>` for an in-container caller. `host-operator` sentinel for a host CLI invocation (FEAT-009 inheritance). |

### Lifecycle states

```text
[non-existent]
    │
    │ route add
    ▼
[enabled=1] ────── route disable ──────▶ [enabled=0]
    ▲                                       │
    │                                       │
    └────────────── route enable ───────────┘
    │
    │ route remove (from either state)
    ▼
[non-existent]
```

The routing worker processes only rows with `enabled=1`. The
cursor `last_consumed_event_id` advances only via the worker's
per-event transaction; CLI operations do not touch the cursor.

### Audit-trail mapping

| Lifecycle event | Audit event type (FR-035) |
|---|---|
| `route add` succeeds | `route_created` |
| `route enable` flips `enabled 0 → 1` | `route_updated` (with `change: {enabled: true}`) |
| `route disable` flips `enabled 1 → 0` | `route_updated` (with `change: {enabled: false}`) |
| `route enable` on already-enabled (no-op) | NONE (FR-009 idempotent) |
| `route disable` on already-disabled (no-op) | NONE (FR-009 idempotent) |
| `route remove` succeeds | `route_deleted` |
| Worker per-event evaluation: enqueue path taken | `route_matched` |
| Worker per-event evaluation: skip path taken | `route_skipped` (with closed-set `reason`) |

## 2. SQLite extension: `message_queue` (FEAT-009 table)

FEAT-010 adds three columns + one partial UNIQUE index to the
existing FEAT-009 `message_queue` table. No existing column is
modified.

```sql
ALTER TABLE message_queue
  ADD COLUMN origin TEXT NOT NULL DEFAULT 'direct';
ALTER TABLE message_queue
  ADD COLUMN route_id TEXT NULL;
ALTER TABLE message_queue
  ADD COLUMN event_id INTEGER NULL;

CREATE UNIQUE INDEX idx_message_queue_route_event
  ON message_queue(route_id, event_id) WHERE origin = 'route';
```

### Column-level invariants

| Column | Invariant |
|---|---|
| `origin` | `'direct'` (FEAT-009 direct-send path) OR `'route'` (FEAT-010 routing worker). Defaults to `'direct'` for backward compatibility (FR-029). |
| `route_id` | `NULL` when `origin='direct'`. References `routes.route_id` when `origin='route'`. No FK constraint (orphan references are intentional per FR-003: removing a route preserves queue-row history). |
| `event_id` | `NULL` when `origin='direct'`. References `events.event_id` when `origin='route'`. No FK constraint. |

### Partial UNIQUE index rationale

The constraint `UNIQUE(route_id, event_id) WHERE origin = 'route'`
enforces FR-030: at most one queue row per `(route_id, event_id)`
pair, defense-in-depth against duplicate routing (the primary
guard is the transactional cursor-advance-with-enqueue per
FR-012). The partial clause keeps direct-send rows
(`origin='direct'`, `route_id=NULL`, `event_id=NULL`) out of the
constraint and sidesteps any SQLite NULL-handling quirk in
plain UNIQUE indexes.

### Effect on `queue.list` socket method

The FEAT-009 `queue.list` socket method gains one optional
keyword-only parameter `origin_filter: Literal['direct', 'route']
| None` (default `None` → no filter). The SELECT becomes:

```sql
SELECT * FROM message_queue
WHERE (:origin_filter IS NULL OR origin = :origin_filter)
ORDER BY enqueued_at, message_id;
```

The CLI exposes this via `agenttower queue --origin
{direct|route}` (FR-033).

## 3. SQLite schema-version bump v7 → v8

`state/schema.py` adds `_apply_migration_v8(conn)` and bumps
`CURRENT_SCHEMA_VERSION = 8`. The migration is idempotent under
the existing `_apply_pending_migrations` framework: a cold-start
daemon at schema v7 will apply only `_apply_migration_v8`; an
interrupted prior migration (column added, index not yet created)
will resume because each `CREATE` uses `IF NOT EXISTS` and each
`ALTER TABLE ADD COLUMN` is guarded by a `PRAGMA table_info`
existence check.

```python
def _apply_migration_v8(conn: sqlite3.Connection) -> None:
    cur = conn.cursor()

    # Step 1: routes table
    cur.execute("""
        CREATE TABLE IF NOT EXISTS routes (
          route_id              TEXT NOT NULL PRIMARY KEY,
          event_type            TEXT NOT NULL,
          source_scope_kind     TEXT NOT NULL,
          source_scope_value    TEXT NULL,
          target_rule           TEXT NOT NULL,
          target_value          TEXT NULL,
          master_rule           TEXT NOT NULL,
          master_value          TEXT NULL,
          template              TEXT NOT NULL,
          enabled               INTEGER NOT NULL DEFAULT 1,
          last_consumed_event_id INTEGER NOT NULL DEFAULT 0,
          created_at            TEXT NOT NULL,
          updated_at            TEXT NOT NULL,
          created_by_agent_id   TEXT NULL
        )
    """)
    cur.execute("""
        CREATE INDEX IF NOT EXISTS idx_routes_created_at_route_id
          ON routes(created_at, route_id)
    """)

    # Step 2: extend message_queue (idempotent column adds)
    existing_cols = {row[1] for row in cur.execute(
        "PRAGMA table_info(message_queue)").fetchall()}
    if "origin" not in existing_cols:
        cur.execute(
            "ALTER TABLE message_queue "
            "ADD COLUMN origin TEXT NOT NULL DEFAULT 'direct'")
    if "route_id" not in existing_cols:
        cur.execute(
            "ALTER TABLE message_queue ADD COLUMN route_id TEXT NULL")
    if "event_id" not in existing_cols:
        cur.execute(
            "ALTER TABLE message_queue ADD COLUMN event_id INTEGER NULL")
    cur.execute("""
        CREATE UNIQUE INDEX IF NOT EXISTS idx_message_queue_route_event
          ON message_queue(route_id, event_id) WHERE origin = 'route'
    """)

    # Step 3: schema version
    cur.execute("UPDATE schema_version SET version = 8")
    conn.commit()
```

## 4. Per-process in-memory state

The routing worker and heartbeat thread share a small in-memory
state object. It is **not** persisted; all counters reset to zero
on daemon restart (FR-038 semantics: "since daemon start").

```python
@dataclass
class _SharedRoutingState:
    cycles_since_last_heartbeat:        int = 0
    events_consumed_since_last_heartbeat: int = 0
    skips_since_last_heartbeat:         int = 0

    # Cumulative since daemon start, exposed via agenttower status
    events_consumed_total:              int = 0
    skips_by_reason:                    dict[str, int] = field(default_factory=dict)
    last_routing_cycle_at:              str | None = None  # ISO-8601 ms UTC

    # Worker degraded state (FR-051). Set TRUE by the worker on a
    # transient internal error (SQLite lock, RoutingDegraded) that
    # prevents cursor advance; cleared on the next successful cycle.
    # Read by heartbeat (mirrored into the `degraded` field of each
    # `routing_worker_heartbeat` entry) and by `agenttower status`
    # (exposed as the `routing_worker_degraded` JSON field).
    routing_worker_degraded:            bool = False

    # Audit-buffer overflow counter (R14). Incremented when the
    # bounded `_pending_audit_buffer` (deque maxlen=10_000) rolls over
    # and drops an oldest entry. Exposed via status for operator
    # visibility into sustained JSONL-write failures.
    audit_buffer_dropped:               int = 0

    lock: threading.Lock = field(default_factory=threading.Lock)


# The status JSON's `degraded_routing_audit_persistence` boolean is
# NOT a field on _SharedRoutingState — it is derived at status-read
# time from the audit buffer:
#
#   degraded_routing_audit_persistence = len(_pending_audit_buffer) > 0
#
# This keeps the two health signals (worker state vs audit-write
# state) independent: a healthy worker may still flag audit
# persistence degraded if events.jsonl is briefly unwritable, and a
# degraded worker may have a fully-flushed audit buffer.
```

The worker mutates these under `lock`. The heartbeat thread reads
+ resets the `*_since_last_heartbeat` counters under `lock`. The
`agenttower status` handler reads the remaining fields under
`lock`.

## 5. Audit buffer (in-memory, bounded)

```python
_pending_audit_buffer: collections.deque[dict] = deque(maxlen=10_000)
```

Populated when `events.writer.append_event` raises
`AuditAppendFailure`. Drained by the worker at the top of each
cycle (retry every pending entry, drop on success). On overflow
(deque rolls over), increment
`_SharedRoutingState.audit_buffer_dropped` and write one
`audit_buffer_overflow` log line.

The buffer also exposes a single read-only helper
`has_pending() -> bool` (returns `len(_pending_audit_buffer) > 0`)
that the `agenttower status` handler uses to compute the
`degraded_routing_audit_persistence` JSON field — see the comment
under `_SharedRoutingState` for the rationale on keeping
worker-degradation and audit-persistence-degradation as separate
signals.

## 8. Configuration knobs (in-memory, CLI-arg sourced)

The three routing-worker knobs are sourced from `agenttowerd`
startup CLI arguments and held in memory only:

- `--routing-cycle-interval-seconds` (default `1.0`, bounds `[0.1, 60]`)
- `--routing-batch-size` (default `100`, bounds `[1, 10000]`)
- `--routing-heartbeat-interval-seconds` (default `60`, bounds `[10, 3600]`)

Knobs are NOT persisted in SQLite in MVP. Operators set them per
daemon process. A `routing_config` table is a forward-compatible
addition for a follow-up feature.

## 9. Migration rollback & concurrency

FEAT-010's v7→v8 schema migration is **forward-only**. Downgrading
from v8 to v7 is not supported by the MVP CLI; rolling back
requires a separate schema-version downgrade tool (out of scope
for FEAT-010).

The migration runs inside daemon startup under the exclusive
SQLite write lock (per FEAT-001's `_apply_pending_migrations`);
concurrent daemon launches are prevented by FEAT-001's PID-file
lock. CLI calls during migration return the standard "daemon not
ready" socket error.

## 10. routing_worker_degraded entry/exit conditions

The `routing_worker_degraded` flag transitions per these rules
(read by `status` socket method + mirrored into every heartbeat):

**Enters degraded (`False → True`)** when ANY of the following
occurs during `_process_route_batch`:
- `sqlite3.OperationalError(database is locked)` raises and the
  transaction rolls back (FR-013 / contracts/error-codes.md §4
  `routing_sqlite_locked`).
- An internal `RoutingDegraded` exception raised by a sub-routine
  (e.g., agents-service unavailable).
- An unexpected exception inside `template.render_template`
  (`routing_internal_render_failure`).
- The audit JSONL buffer is non-empty at cycle start (drain
  required).

**Exits degraded (`True → False`)** at the first cycle that
completes WITHOUT any of the above conditions AND with an empty
audit buffer.

## 11. Runtime fields persistence model (route show)

The `runtime` sub-object in `route show --json` (FR-047) is
sourced as follows:

- `last_routing_cycle_at` — from `_SharedRoutingState.last_routing_cycle_at` (in-memory; resets on daemon restart).
- `events_consumed` — derived at read time from
  `routes.last_consumed_event_id` minus the
  `last_consumed_event_id_at_creation` (a column NOT in v8;
  computed instead as "current cursor minus 0" — operators
  interpret this as "events consumed since route creation" only
  when the daemon has not been restarted; resets to current-cursor
  on restart). Acceptable inaccuracy for MVP.
- `last_skip_reason` — from `_SharedRoutingState.last_skip_per_route[route_id]` (in-memory dict; resets on restart).
- `last_skip_at` — same dict, paired timestamp.

Persistence to SQLite for cross-restart continuity is a
forward-compatible follow-up.
